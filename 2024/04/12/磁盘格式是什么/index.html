磁盘格式是什么

发表于 2024-04-12 分类于 日常考据

闲言碎语

使用MAC读U盘或者移动硬盘，基本都会遇到只能读不能写或者读都不能读的情况，大概了解是因为磁盘格式的问题导致，也知道下载Mounty软件来解决问题。但是里边具体的原因以及解决的办法是啥都不太清楚。

遂查资料

磁盘格式是什么

我们说的磁盘格式是指磁盘文件系统，磁盘文件系统是一种设计用来利用数据存储设备来保存计算机文件的文件系统，最常用的数据存储设备是磁盘驱动器，可以直接或者间接地连接到计算机上。例如：文件分配表（FAT12、FAT16、FAT32、exFAT）、New Technology File System、分层文件系统HFS、HFS Plus、延伸文件系统（ext1、ext2、ext3、ext4）、ODS、btrfs、XFS、UFS、ZFS。

也就是说，磁盘格式 = 磁盘文件系统 = 文件系统中的一种。

文件系统是什么

文件系统是一种存储和组织计算机数据的方法，它使得对其访问和查找变得容易。

文件系统使用文件和树形目录的抽象逻辑概念代替了硬盘和光盘等物理设备使用数据块的概念，用户使用文件系统来保存数据不必关心数据实际保存在硬盘（或者光盘）的地址为多少的数据块上，只需要记住这个文件的所属目录和文件名。在写入新数据之前，用户不必关心硬盘上的那个块地址没有被使用，硬盘上的存储空间管理（分配和释放）功能由文件系统自动完成，用户只需要记住数据被写入到了哪个文件中。

文件系统通常使用硬盘和光盘这样的存储设备，并维护文件在设备中的物理位置，实现了数据的存储、分级组织、访问和获取等操作。

文件系统将设备中的空间划分为特定大小的块（或者称为簇），一般每块512字节。数据存储在这些块中，大小被修正为占用整数个块。由文件系统软件来负责将这些块组织为文件和目录，并记录哪些块被分配给了哪个文件，以及哪些块没有被使用。

其他文件系统

除了磁盘文件系统外，还有用于CD、DVD、蓝光盘的光盘文件系统（ISO 9660和UDF），用于存储卡、U盘等的闪存文件系统（JFFS2和YAFFS，现在一般使用exFAT），用于数据库的数据库文件系统，用于远程挂载的网络文件系统。

常见文件系统类型

FAT

文件分配表（File Allocation Table），1977年由比尔·盖茨和马斯·麦当劳为了管理磁盘而发明，并在1980年被添·彼得逊的86-DOS操作系统采用。

FAT文件系统考虑当时电脑性能有限，所以未被复杂化，因此几乎所有PC的操作系统都支持。这特性使它成为理想的软盘和存储卡文件系统，也适合用作不同操作系统中的资料交流。

但FAT有一个严重的缺点：当文件删除后写入新资料，FAT不会将文件整理成完整片段再写入，长期使用后会使文件资料变得逐渐分散，而减慢了读写速度。碎片整理是一种解决方法，但必须经常磁盘碎片整理来保持FAT文件系统的效率。

  • FAT12: 最初的FAT版本由微软在1980年代初为DOS操作系统开发。FAT12被设计用于管理非常小的磁盘和简单的存储结构，最多支持容量为16MB的磁盘。
  • FAT16: 随着技术的发展和存储需求的增加，FAT16在1984年随DOS 3.0推出，最大支持2GB的磁盘。
  • FAT32: 为了克服FAT16在磁盘容量上的限制，FAT32于1996年随Windows 95 OSR2推出，单个文件的大小限制为4GB，而卷的最大大小通常被操作系统限制在32GB。
  • exFAT（Extended File Allocation Table） - 微软2006年发布，旨在超越FAT32的文件大小和卷大小限制，特别是针对高容量的存储设备，如USB闪存驱动器和SD卡。支持单个文件大小超过4GB（理论上可以达到16EB - Exabytes），卷大小也可以非常大，理论上支持到16EB。由于它是较新的系统，老旧设备可能不支持，老旧设备还需要使用FAT32。

NTFS

新型文件系统（New Technology File System），NTFS是微软在1993年随Windows NT 3.1推出NTFS，以满足更高的数据安全性、更大的文件和卷支持需求。NTFS支持文件压缩、加密和权限控制等高级特性，是Windows系统中最常用的文件系统。

HFS

分层式文件系统（Hierarchical File System），苹果公司为Macintosh计算机在1985年设计的文件系统，被称为Mac OS Standard。它为文件和文件夹提供了分层的组织结构。

HFS+（也称为Mac OS Extended）: 1998年推出，解决了原始HFS文件系统的许多限制，如文件名长度和最大卷大小，支持更大的文件和Unicode编码。

EXT

延伸文件系统（Extended File System），1992年，为Linux内核开发的第一个文件系统，被设计来克服当时文件系统的限制。

  • EXT2: 1993年推出，成为Linux的首选文件系统，提供了良好的性能和稳定性。
  • EXT3: 2001年，为EXT2添加了日志功能，提高了数据恢复的能力。
  • EXT4: 2008年，进一步增强了性能，支持更大的单个文件和总存储容量，改进了磁盘空间的分配。

APFS

苹果文件系统（Apple File System），苹果公司为提高效率和安全性，于2017年随iOS 10.3和macOS High Sierra推出，专为固态驱动器和先进的数据安全特性设计，支持快照、克隆和加密等功能。

支持情况

文件系统 Windows macOS Linux
FAT32 读/写 读/写 读/写
exFAT 读/写 读/写 读/写 (需安装额外软件支持)
NTFS 读/写 读 (默认) / 写 (需第三方软件) 读/写 (需安装额外软件支持)
EXT4 不支持 (需第三方软件) 不支持 (需第三方软件) 读/写
APFS 不支持 (需第三方软件) 读/写 读 (有限支持，需安装额外软件)

Windows:

  • 原生支持FAT32、exFAT和NTFS；
  • 不支持EXT4，需要通过第三方软件，如Ext2Fsd；
  • 不支持APFS，需要通过第三方软件，如MacDrive来访问。

macOS:

  • 原生支持FAT32、exFAT和APFS。
  • 对NTFS的写入支持需要第三方软件，如NTFS for Mac、Mounty。
  • 不支持EXT4，需要第三方软件来访问，如macFUSE。

Linux:

  • 原生支持FAT32、EXT4。
  • 对NTFS的完整支持（包括读写）通常需要安装额外的软件包如ntfs-3g。
  • 对exFAT的支持也需要安装额外软件，如FUSE exFAT。
  • APFS的支持是有限的，主要是只读，并且需要额外的软件支持，如apfs-fuse。
